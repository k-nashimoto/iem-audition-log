# IEM 試聴ログ (IEM Audition Log)

ハイエンドIEM（イヤモニ）を試聴・比較し、評価を蓄積するための個人向けWebアプリ。
スマホのブラウザ／ホーム画面から開いて、店頭試聴でその場に記録できる。

## 特長

- 機種ごとに **日付・機種名・ソース** を付けて試聴を記録
- 8つの確認軸（装着 / ボーカルの艶 / キラキラvs刺さり / 低域 / 包まれる没入 / 生音感 / 強奏 / 解像）に沿って各曲を **◎○△✕** で評価＋メモ
- 記録は **ブラウザに自動保存**（localStorage）。一覧から後で再編集
- 全データの **JSON 書出/読込**（バックアップ・端末移行）
- 単一HTML・外部依存なし・**オフライン動作**。モバイルファースト

## 使い方

1. `iem-audition-log.html` をブラウザで開く（またはGitHub Pagesの公開URL）。
2. 「＋ 新規試聴を記録」で機種名・日付・ソースを入力。
3. 8カテゴリのチェックリストで各曲を聴きながら ◎○△✕ とメモを記録。
4. 一覧画面から過去の試聴をいつでも再編集・比較。

## ホーム画面に追加（iPhone）

GitHub Pages の公開URLを Safari で開く → 共有 → 「ホーム画面に追加」。
以後はアイコンから全画面で起動、初回読込後はオフラインでも開ける。

## デプロイ（GitHub Pages）

`main` ブランチの `/ (root)` を Pages のソースに設定。
`iem-audition-log.html` を更新して push すると自動で本番反映される。

## 開発

- 単一ファイル `iem-audition-log.html`（HTML / CSS / vanilla JS、ビルド工程なし）。
- データは `localStorage` キー `audition-log-v1`。
- 設計コンテキストは `CLAUDE.md`、拡張計画は `ROADMAP.md` を参照。
