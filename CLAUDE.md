## 1. プロジェクト概要

オーナー（梨本健太）が、ハイエンドIEM（イヤモニ）を試聴・比較し、評価を蓄積するための個人ツール。
現状は単一HTMLの「試聴ログ」アプリ（`iem-audition-log.html`）。GitHub Pages でホストし、iPhoneのホーム画面から起動して店頭試聴で使う。

最終目標（段階的に実現。詳細は `ROADMAP.md`）:
1. 試聴データの蓄積・機種横断比較のシステム化
2. 世の中のIEM情報の検索・集約
3. Qobuz / Apple Music と連携した試聴プレイリストの編集・再生

## 2. オーナーの音の好み（最重要・設計の基準）

機能やデフォルト、評価軸はこのプロファイルに合わせること。
オーナーの音の好み詳細は `PROFILE.md` を参照。

- **低域**: 存在感はしっかり、ただし量は出しすぎない。締まってグルーヴを生む「量より質」。
- **高域**: キラキラした輝き＋高い解像度。ただし**刺さらない**こと（サ行・8〜10kHzのピークに敏感）。
- **中域/ボーカル**: 生々しさと「艶」。emotional realism を重視。
- **空間**: 「包まれる没入感」が最重要の決定軸（映画サントラ・オーケストラの大編成スケール）。
- **生音感**: 楽器の実在感。VE10は「温度・実体（A方向）」、Kronosは「空気・余韻・倍音（B方向）」と整理。
- 嫌うもの: clinical coldness（無機質）、boomy bass、treble spike。

ジャンル: J-pop / K-pop / 映画・ゲームOST / オーケストラ。

## 3. 機材・試聴の前提

- **リファレンスIEM**: Vision Ears VE10（所有。音数少なめの女性ボーカルが最高。高感度で iFi アッテネーター併用、Effect Audio A×C ケーブル 4.4mm）。
- **ソース(DAP)**: Cayin N7+（クラスA基準。クラスA↔AB切替・ゲイン段あり）。
- **2台目の最有力候補**: Noble Kronos（約¥693,000）。映画/オーケストラの「包まれる没入」用途。VE10と補完関係。
- **ストリーミング**: Apple Music（メイン）、Qobuz（連携検討対象）。
- 試聴時の鉄則: まず深いシール確保（Kronosは左Sサイズ・右Mサイズで安定）→ 音量は数値でなく聴感で揃える → アッテネーターは付けたまま。

## 4. 試聴リファレンス（8カテゴリ + 番外 / 試聴リストv2.2）

アプリのチェックリストは Apple Music「試聴用プレイリスト」v2.2 に準拠。各曲を1カテゴリに割当（1曲1評価）。メインカテゴリは 00–07 の8つで不変（既存記録と互換）。

- `00` 装着・シール確認
- `01` ボーカルの艶・生々しさ【最重視】（サブ観点 01a 女性 / 01b 男性）
- `02` キラキラ vs 刺さり【重要・感度】
- `03` 締まったグルーヴ低域
- `04` 包まれる没入・空間スケール【決定軸】（サブ観点 04a ホール残響 / 04b 音壁 / 04c コーラス包囲。◎は3タイプ○以上が条件）
- `05` 生音感（サブ観点 05a 温度・実体(A) / 05b 空気・余韻(B)）
- `06` ダイナミクス・強奏（サブ観点 06a 立ち上がり / 06b 飽和耐性 / 06c 漸強）
- `07` 解像・分離・一体感
- `EX` 番外（愛着枠。序曲のマーチ。評価軸ではない）

- 評価記号: ◎=理想 / ○=良 / △=惜しい / ✕=合わない。
- サブ観点は各曲の**タグ**として表現（曲評価＝サブ観点評価）。カテゴリ評価は曲評価の平均で、別枠の手動メインスコアは持たない。
- **★店頭コア7**は `core` フラグ、詳細画面に「コアのみ表示」トグルあり。
- 各曲の `id` は**安定slug**（例 `first-love`）。rating/notes/openMemo はこの id をキーにする（旧・位置ベース `01-0` からは `OLD_ID_MAP` で冪等移行）。
- **曲リストと評価は分離**：`TRACKS`（曲リスト＝source of truth、`cat` で所属カテゴリ参照）と `CATEGORIES`（カテゴリ定義）を持ち、`CATS` はその結合ビュー。曲の改訂は `TRACKS` を編集するだけで、評価は id 参照なので壊れない。改訂時は `CATALOG_VERSION` を上げる。
- **孤児評価**（現行カタログに無い曲の評価）は集計から除外しつつ保持（`CATALOG_IDS` で判定、詳細画面に件数注記）。

## 5. 現状の技術構成

- **単一ファイル**: `iem-audition-log.html`（HTML+CSS+vanilla JS、ビルド工程なし、外部依存なし＝オフライン動作）。
- **保存**: ブラウザの `localStorage`、キー `audition-log-v1`。Claude.ai アーティファクト内では動かないが、ブラウザ/GitHub Pages では正常動作。
- **データモデル**: `store = { sessions: [], activeId, compareIds:[], compareMakers?, cmpMode }`。各 session = `{ id, maker, iem, date, source, app, conn, codec, cable, catalogVersion, createdAt, ratings:{trackId}, notes:{trackId}, openMemo:{} }`。trackId は各曲の**安定slug**（例 `first-love`）。旧・位置ベース `"<カテゴリNo>-<index>"` は `OLD_ID_MAP` で移行。`maker`（メーカー）は機種名と別枠でメーカー別集計の基盤。
- **画面**: 一覧ビュー（機種カード=メーカー・機種・日付・進捗・◎数）／詳細ビュー（メーカー・機種・日付・ソース・再生アプリ・接続方式編集＋カテゴリ別チェックリスト＋コアのみ表示）／比較ビュー（機種別／メーカー別モード、カテゴリ×平均のマトリクス、メーカー別は得意/注意の傾向）。
- **書出/読込**: 全データの JSON エクスポート・インポート（端末移行・バックアップ用）。
- **デプロイ**: GitHub Pages。リポジトリ `k-nashimoto/iem-audition-log`、`main` / `(root)`。
  公開URL: `https://k-nashimoto.github.io/iem-audition-log/iem-audition-log.html`
  → **commit & push すれば自動で本番反映**。

## 6. デザインシステム / 規約

- ダーク基調のプレミアム志向。二金属アクセント:
  - ゴールド `--gold` = 最重視タグ・◎評価
  - スチール `--steel` = 構造・○評価
  - シアン `--cyan` = 決定軸タグ
  - 警告色 `--caution` = △ / 重要・感度タグ、`--bad` = ✕
- 数値・ラベルは等幅（instrument readout 風）、本文は日本語システムフォント。
- モバイルファースト、タップ領域は大きめ、`prefers-reduced-motion` 尊重。

## 7. 作業上の約束（重要）

- **オフライン動作を壊さない**（外部CDN/フォント/APIに無断で依存しない。フェーズ3で連携を入れる際は明示的に設計）。

- 著作権: 歌詞など著作物の本文を埋め込まない（曲は曲名・アーティストの参照のみ）。
